8  * Energine is a content management system which allows to support web-applications (including websites) of any level of complexity. Energine is based on Energine CMF â€” a power full toolkit for web-application development using XML/XSLT transformations.
9  *
10  * Main features of Energine are:
11  * - Multi-language support. Energine supports unbounded quantity of languages with ability to translate not only content of a site, but buttons, emails, captions too.
12  * - %User's access delimitation. %User's access control system allows to edit user's rights to access and edit different parts of a website.
13  * - Visual text editor. A built in WYSIWYG (what you see is what you get) editor is a handy tool to edit web site's content and preview it.
14  * - Files. Common file storage allows to use one method to work with files in forms and with a help of text editor.
15  * - Structure site management. Web site's structure represented as a tree. %User can add, edit and delete it's nodes to modify parts of a site.
16  * - Shop module. Additional module which allows to create and use eShop.
17  *
18  * @section foreword Forewords.
19  *
20  * Since Energine 2.11.0 the project structure was changed.
21  * Now the releases of Energine core can be stored in separate directory (for example <tt>/var/www/energine</tt>).
22  * The projects are now no longer pull the kernel and modules over svn:externals, they connect required versions of core and external modules over configuration file.
23  *
24  * @section install Installation guide
25  *
26  * -# Create directory for core (for example <tt>/var/www/energine</tt>)
27  * @code
28 mkdir /var/www/energine
29 @endcode
30  * -# Download and extract Energine release
31  * - from Github https://github.com/energine-cmf/energine/releases
32  * - or get clone of our repository
33  * @code
34 cd /var/www
35 git clone git@github.com:energine-cmf/energine.git
36 @endcode
37  * -# Copy default starter
38  * @code
39 cd /home/username/projects
40 cp -r /var/www/energine/starter www.mynewsite.com
41 @endcode
42  * -# Create database.
43  * <ol>
44  * <li>Import base site structure from <tt>sql/starter.structure.sql</tt>
45  * <li>Import stored procedures from <tt>sql/starter.routines.sql</tt>
46  * <li>Import system data from <tt>sql/starter.data.sql</tt>
47  * </ol>
48  * -# Configurations.
49  * <ol>
50  * <li>Copy configuration file from <tt>configs/system.config.default.php</tt> to <tt>configs/system.config.username.php</tt>.
51  * <li>Make symlink from <tt>configs/system.config.username.php</tt> to <tt>htdocs/system.config.php</tt>.
52  * <li>Edit configuration file:
53  * <ol>
54  * <li>Set directory of the Energine core
55  * <li>Set correct database configurations
56  * <li>Set site domain
57  * <li>Set the modules, which will be accessed from the site (full path for each module)
58  * <li>Set your E-Mail
59  * </ol>
60  * </ol>
61  * -# Configure web-server: <tt>nginx + php_fpm</tt> or <tt>apache2 + mod_php</tt>
62  * -# @c nginx
63  * <ol>
64  * <li>Base config file is in <tt>jambalaya/.nginx.conf.example</tt>. You can copy this config into the directory @c conf.d of your nginx as conf file (for example <tt>www.mysite.com.conf</tt>).
65  * <li>Set your connection to @c php-fpm in <tt>upstream php-fpm</tt> block
66  * <li>Set correct @c server_name and listen port in @c server block
67  * <li>Set absolute path of the project into <tt>$www_folder</tt> variable in server block
68  * <li>Set permissions to write to the directory <tt>htdocs/uploads</tt>.
69  * </ol>
70  * -# @c apache2
71  * <ol>
72  * <li>Configure your VirtualHost if it is not yet done.
73  * <li>Base config file is in <tt>jambalaya/.htaccess</tt>. Copy it to <tt>www.mysite.com/htdocs/</tt>
74  * <li>Set the single directive @c RewriteBase. If the project is placed in some folder, not in site core, then the directive should be: <tt>RewriteBase /~username/some/folder/</tt>. If it placed in site core then: <tt>RewriteBase /</tt>
75  * <li>Set permissions to write to the directories: <tt>htdocs, htdocs/uploads, htdocs/core/modules</tt>.
76  * </ol>
77  * -# Run http://www.your.project.address/setup/ If all is ok - you will have a working system copy based on Energine engine.
78  * -# Develop your site
79  * -# If you use @c apache2, remove the permissions to write to the directories: <tt>htdocs, htdocs/core/modules</tt>.
80  * -# Turn off debug mode in configuration file.
81  */
